# Web-based Pay Calculator

## Context

The existing pipeline (`scripts/generate_post_increment_payslip.py`,
`scripts/generate_promotion_payslip.py`) projects a hypothetical payslip,
but only for the one employee whose `mar.csv` / `jan.csv` baselines are
checked in, and only by editing and re-running Python. The user wants a
browser UI where they can pick a scenario (annual increment vs
promotion), pick any grade, pick any current basic-pay stage, and see
the projected payslip — without touching code.

Outcome: a single self-contained `pay_calculator.html` at the repo root,
generated by a new script in `scripts/`, that reproduces the same
calculations as the two Python projection scripts but parameterised by
grade and starting stage.

## Approach

One new script + one generated HTML deliverable. No backend.

### New file: `scripts/generate_calculator_html.py`

Reads the canonical CSV inputs and bakes them into a single HTML page:

- `PAYSCALE23.csv` → JS object `PAYSCALE` keyed by grade with
  `{min, increment, max, stages: number[20]}`.
- `jan.csv` and `mar.csv` → JS objects `JAN_DEDUCTIONS`, `MAR_EARNINGS`
  for carry-over fields (FUEL, ENTERTAINMENT, CATEGORIZATION ALL.,
  ADHOC RELIEF-2023 20%, ADHOC RELIEF 2024 30%, GRP INSURANCE,
  BENEVOLENT FUND, WELFARE FUND, AL-SHIFA TRUST, ATC GUILD CONTR. FUND,
  CAR ADVANCE, GPF ADVANCE).
- Tax slabs from `tax_rules.md` mirrored as a JS array (same shape as
  `SLABS` in `scripts/calculate_income_tax.py`).

The script is the analogue of `convert_payscale23.py` — read source,
emit a deliverable. Mirrors the loaders already used in
`generate_post_increment_payslip.py` (`load_grade_row`, `load_slip`).

### Generated file: `pay_calculator.html` (repo root)

Single HTML file (no external assets, no network calls). Layout:

1. **Mode buttons** (radio-style toggle): `Annual Increment` /
   `Promotion`.
2. **Grade `<select>`** populated from `PAYSCALE` keys (SG-01 … EG-12).
3. **Current Basic `<select>`** populated with the 20 stage values for
   the chosen grade (re-populates on grade change). For Promotion mode,
   the next-grade lookup happens automatically — current grade still
   drives the starting basic.
4. **Calculate** button.
5. **Result panel** rendered as three columns mirroring
   `render_payslip_pdf.py`: Earnings | Deductions | Misc/Summary, with
   GROSS / TOTAL DEDUCTIONS / NET PAY footer, plus a notes block
   showing the basic transition (e.g. `163,430 → 171,600`) and the
   annualised tax base used.

### JS calculation logic (must match the Python scripts exactly)

Per `pay_rules.md`:

- **Annual Increment**: `new_basic = stages[idx(current) + 1]` on the
  same grade row. If already at stage 20, surface an error message in
  the result panel.
- **Promotion**: find the next grade alphabetically/sequentially after
  the current one (use the order of keys in `PAYSCALE`); within that
  grade pick the lowest stage strictly greater than `current_basic`;
  add one increment of the new grade. (Identical to the rule in
  `pay_rules.md` and the assertion in `generate_promotion_payslip.py`.)
- **Earnings (basic-linked)** rounded with `Math.round`:
  HOUSE RENT = RATING = 0.80 × basic; UTILITY = MEDICAL = 0.25 × basic;
  ATCL ALLOWANCE = 0.20 × basic; M/CYCLE MAINT. = `grade.increment`.
- **Frozen / carry-over earnings** taken from `MAR_EARNINGS` baked
  into the page: FUEL, ENTERTAINMENT, CATEGORIZATION ALL.,
  ADHOC RELIEF-2023 20%, ADHOC RELIEF 2024 30%.
- **Deductions**: GP FUND = round(0.08 × (min + max) / 2) of the
  *target* grade (changes only on promotion); GRP INSURANCE,
  BENEVOLENT FUND, WELFARE FUND, AL-SHIFA TRUST,
  ATC GUILD CONTR. FUND, CAR ADVANCE, GPF ADVANCE all from
  `JAN_DEDUCTIONS`.
- **Income tax**: JS port of `annual_tax()` from
  `scripts/calculate_income_tax.py`; `monthly_tax =
  round(annual_tax(sum(earnings) × 12) / 12)`.
- Gross = sum(earnings); Total Ded = sum(deductions); Net = Gross − Total Ded.

### Critical files to read / mirror

- `scripts/generate_post_increment_payslip.py` — formulas, carry-over
  source, tax recompute pattern.
- `scripts/generate_promotion_payslip.py` — promotion fixation rule,
  GP FUND on new grade.
- `scripts/calculate_income_tax.py` — slab function to port to JS.
- `scripts/render_payslip_pdf.py` — three-column visual layout to
  mimic in the HTML result panel.
- `pay_rules.md`, `tax_rules.md` — authoritative formulas / slabs.
- `PAYSCALE23.csv`, `jan.csv`, `mar.csv` — embedded data sources.

## Files Touched

- **New**: `scripts/generate_calculator_html.py`
- **New (generated)**: `pay_calculator.html` (repo root, generated

No existing file is modified. No new Python deps (stdlib `csv` only).

## Verification

1. Run `python scripts/generate_calculator_html.py` — confirms it reads
   the three CSVs and writes `pay_calculator.html`.
2. Open `pay_calculator.html` in Firefox.
3. **Annual-increment regression**: pick `EG-04`, current basic
   `163,430`, mode `Annual Increment`, click Calculate. Compare against
   `jul_post_increment.csv` — every earning, deduction, and the net
   pay must match line-for-line.
4. **Promotion regression**: pick `EG-04`, current basic `179,770`,
   mode `Promotion`. Compare against `may_promotion.csv` — basic must
   come out to `194,080` and totals must match.
5. **Tax slab spot-check**: enter a scenario whose annualised gross
   crosses 4,100,000 and confirm the JS tax matches
   `python scripts/calculate_income_tax.py <annual>`.
6. **Edge case**: pick stage 20 in Annual Increment mode — calculator
   must show a clear "already at top of grade" message instead of
   silently producing the same basic.
